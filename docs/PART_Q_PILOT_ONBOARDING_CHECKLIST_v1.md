# Part Q - Pilot Onboarding Checklist v1

Status: active Part Q execution document.

This checklist controls how approved pilot participants enter the Supplier Seed pilot.

## Purpose

Every participant should follow the same onboarding flow so the pilot remains controlled, auditable, and reversible.

## Onboarding Stages

### 1. Invitation

Confirm:
- organization name
- contact person
- pilot group
- expected role
- approved pilot boundary

### 2. Registry Entry

Create or confirm the participant registry record before access is enabled.

Required:
- participant ID
- organization
- contact
- pilot group
- assigned role
- initial status

### 3. Terms Acceptance

Before active use, record:
- terms version
- acceptance timestamp
- accepting actor
- related organization

### 4. Access Activation

Access may be enabled only after registry and terms checks pass.

Activation should create an audit event.

### 5. Training Walkthrough

Walk through:
- supplier creation
- moderation review
- verification review
- activation review
- denied-action handling
- incident reporting

### 6. Live Pilot Use

The participant may begin controlled live pilot use only after all prior stages are complete.

## Completion Check

Participant onboarding is complete only when:
- registry entry exists
- pilot status is correct
- terms acceptance is recorded
- role is assigned
- access is enabled
- training walkthrough is complete
- audit trail exists

## Failure Handling

If any step fails, keep the participant in Pending Acceptance or Disabled status until corrected.
