# Part Q - Operator Workflow Evidence Checklist v1

Status: active Part Q execution document.

This checklist records whether real pilot users can complete the main Supplier Seed workflows without governance bypass or excessive support.

## Purpose

Part Q must prove that workflows work for real operators, not only in tests.

## Workflows To Evidence

### Supplier Creation

Capture:
- participant
- supplier created
- creation timestamp
- result
- errors or confusion
- audit reference

### Moderation Review

Capture:
- queue entry
- reviewer
- decision
- decision timestamp
- reason or notes
- audit reference

### Verification Review

Capture:
- submitted supplier
- verifier
- verification result
- timestamp
- missing information, if any
- audit reference

### Supplier Activation

Capture:
- supplier ID
- approving actor
- activation status
- timestamp
- required gates confirmed
- audit reference

### Denied Action Handling

Capture:
- actor
- attempted action
- denial reason
- role involved
- timestamp
- audit reference

### Incident Reporting

Capture:
- reporter
- affected workflow
- severity
- summary
- linked support item, if any
- resolution status

## Operator Friction Notes

Track:
- confusing labels
- repeated questions
- unnecessary steps
- delayed decisions
- missing information
- training gaps

## Completion Rule

Workflow evidence is complete only when each core workflow has at least one real pilot example and any issue is linked to a support item, incident, or Part R stabilization finding.
