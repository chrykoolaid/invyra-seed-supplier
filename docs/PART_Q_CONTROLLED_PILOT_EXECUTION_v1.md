# Part Q - Controlled Pilot Execution v1

Status: active next phase after Part P.

Part Q starts the controlled pilot phase for the Supplier Seed System.

This phase keeps the rollout limited, reversible, auditable, and PH-first.

## Objective

Run a small pilot using the locked Part P baseline and verify real supplier workflows with approved users only.

## Allowed Scope

- approved pilot participants
- approved internal operators
- friendly pilot businesses
- controlled supplier entries
- supplier creation
- moderation review
- verification review
- supplier activation
- KPI capture
- incident logging

## Out of Scope

- public launch
- open marketplace rollout
- uncontrolled region rollout
- speculative feature additions
- unapproved bulk imports

## Pilot Access States

- Disabled
- Pending Acceptance
- Active
- Suspended
- Closed

## Acceptance Criteria

Part Q completes only when the pilot group is defined, onboarding is complete, live workflows are exercised, KPI capture is active, incidents are logged, and rollback review has been completed.

## Output

Part Q must produce a Pilot Completion Report before moving to Part R.
