# Part Q - KPI and Incident Tracking v1

Status: active Part Q execution document.

This document defines the minimum evidence needed during the controlled pilot.

## Purpose

The pilot must be measured with real usage evidence, not assumptions.

KPI and incident tracking should show whether the Supplier Seed System is safe, usable, and governed in pilot conditions.

## Minimum KPIs

Track at least:

- active pilot participants
- supplier creation attempts
- successful supplier creations
- moderation queue entries
- moderation decisions
- verification submissions
- completed verifications
- supplier activations
- denied actions
- logged incidents

## Operational Measures

Review:

- creation success rate
- verification completion rate
- moderation processing time
- operator error rate
- incident rate
- repeat support questions

## Incident Fields

Each incident should include:

- incident ID
- date
- reporter
- affected participant
- affected workflow
- severity
- summary
- current status
- root cause, when known
- resolution notes
- follow up action

## Severity Levels

- P1 Critical
- P2 Major
- P3 Minor
- P4 Observation

P1 items block normal pilot continuation until reviewed.

## Weekly Review Output

Each weekly review should summarize:

- usage activity
- KPI movement
- incidents opened
- incidents resolved
- unresolved risks
- continue, pause, or rollback recommendation

## Exit Use

These records feed the Part Q Pilot Completion Report and the Part R stabilization backlog.
